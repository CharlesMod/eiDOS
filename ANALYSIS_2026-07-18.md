# eiDOS — Deep Analysis & SOTA Review (2026-07-18)

> Produced by a two-pronged, adversarially-verified investigation on branch `analysis/fixes-2026-07-18`.
> **Prong A** — an 8-lens code audit (57 agents; each finding re-checked against the real source by an
> independent skeptic). **Prong B** — an 8-domain state-of-the-art web survey mapped against what eiDOS
> already has. Everything below is judged against the operator's stated pillar: **autonomy — persist toward
> a goal without going off-track, going "crazy" (rumination), or hitting a wall and spinning its gears** —
> and the operator's bar: **efficacy**, where *dark (built-but-unwired) functions are defects, not deferred work.*

## TL;DR

1. **The architecture is right; the defects are engineering/activation gaps, not missing design.** The 2025–26
   field converged on eiDOS's founding bet (frozen weights + all learning in external state/glue: CoALA,
   ReasoningBank, ACE, Voyager, Memento). eiDOS is **at or ahead of SOTA** on the autonomy machinery itself —
   the deterministic frustration/rotation gate, block-as-falsifiable-belief, the reserved-exploration
   anti-tunnel slot, scalar-reward-*plus*-verbal-lessons, anti-wireheading self-edit governance, and
   arousal-gated sleep all beat or match what's published.
2. **The autonomy pillar has two real live-path holes** (both HIGH): an impossible self-goal can become
   *immortal* (weak "progress" relieves frustration forever, and the death gate is bypassable via the
   creature's own `objective_block` tool). These nullify the exact 2026-07-13 doom-loop fix they were built for.
3. **Two "dark organ wearing a live flag" defects on the sensory seam** (the operator's explicit concern):
   the **salience gate** computes a full ranked admission every tick that **the core never reads**, and the
   **felt body flickers out of the prompt** on idle ticks (the afferent intake reads the transient mailbox,
   not the retained snapshot).
4. **A memory-continuity HIGH:** with the engram cascade live, **facts learned after boot are never recalled
   by relevance** — the "wakes amnesiac / re-derives what it knows" failure, on the live path.
5. **The three highest-value, lowest-effort moves are internal wiring of things already built** — and Prong A
   and Prong B independently agree on them: wire the dark learning-progress signal into curiosity, add
   decay/eviction to the engram store, and add a ReasoningBank-style strategy memory.
6. **The missing meta-capability is measurement** — there is no external yardstick for "coherent goal-pursuit
   horizon," so no anti-derailment change can be judged. Instrument one.

**Counts:** 47 findings survived adversarial verification (2 refuted) — **5 HIGH, 18 MEDIUM, 24 LOW**.
No finding survived at *critical* (the one nominated was downgraded on verification — good discipline).

## 1. The autonomy scorecard (the pillar)

**Where eiDOS is genuinely strong (keep, don't replace):**
- **Deterministic anti-rabbit-hole control** — per-goal frustration + automatic focus rotation is *more*
  principled than the 2025 industry standard (action-hash window + token circuit-breaker + hard iteration cap).
- **BLOCK-as-falsifiable-belief** — treating a blocker as a hypothesis re-tested as "exposures," released DEAD
  after a cap, is not present in any surveyed agent framework.
- **Reserved exploration slot** in the recall cascade — a built-in anti-tunnel-vision mechanism.
- **Scalar reward (TD/RPE) *coupled with* verbal lessons** — the dominant SOTA line (Reflexion/ExpeL/ACE) is
  verbal-only; the earned-strength bet-ledger with provenance+decay is a running defense against the
  literature's #1 documented failure (self-reinforcing confabulated "lessons").

**Where the pillar actually leaks (fix these):**
- **H1/H2 — the doom-loop gate is bypassable** (weak-progress immortality + block-tool ping-pong). §2.
- **Reward freebie** — a bash command with naturally-varying output (`date`, `uptime`, a growing log, *any*
  async bash — the dispatch ack embeds a fresh PID every tick) pays the full **+0.40 success** with zero
  progress, and can crystallize into a coached "when idle, run &lt;probe&gt;" habit. One reward-hack class over
  from the re-read spiral. §2.
- **Loop detector is arg-vary-defeatable and toothless** — non-bash tool signatures aren't normalized, and its
  only response is an advisory prose plea, not a structural consequence.
- **Backlog-exhaustion soft-stall** — when the whole backlog parks, nothing mints a new commitment; the
  creature idles until a 25-tick thaw cooldown.

## 2. Prong A — HIGH findings (live-path, verified)

### H1 · Impossible self-goal never parks *or* dies — `objectives.py:469` (autonomy)
Weak progress (a single new file in `workspace/home`) **fully relieves frustration** (`FRUST_RELIEF=3`,
unconditional on `progress_strong`) and resets the stall clock. The `EXPOSURE_CAP` death check lives *only*
inside the `frustration >= park_at` branch. So a creature brooding on an impossible goal that mints a cosmetic
file every few ticks keeps frustration near zero → never parks → never blocks → never thaws → the death gate is
**never evaluated**. The content-aware "motif brake" (built to catch a loop that journals one theme through
action tools) is also skipped on weak-progress ticks. *This is exactly "spinning its gears on a wall" — and it
defeats the 2026-07-13 doom-loop fix.*
**Fix:** gate frustration *relief* on `progress_strong` (weak progress gets fractional or zero relief), and/or
add a `ticks_since_STRONG_progress` death clock independent of the frustration threshold.

### H2 · `EXPOSURE_CAP` death gate bypassed by the `objective_block` tool — `objectives.py:482` (autonomy)
Death-on-cap lives only in the frustration-park branch. `objectives.block()` (the model-facing `objective_block`
tool, granted at genesis) sets `state="blocked"` with no cap/death check; the thaw paths increment `exposures`
but nothing consumes that budget outside the one park branch. So the model can **thaw → block → thaw** forever,
never routing through the death gate. (A nap-time archive backstop exists but is evaded by the same weak-progress
trick as H1.)
**Fix:** move the exposure/death evaluation into `_thaw` — the single choke point every re-activation passes
through — and count model-driven `objective_block` re-parks toward the budget.

### H3 · Post-boot facts are never recalled by relevance — `eidos.py:755` (context efficacy)
`memory_manager_enabled` is **true** in the effective config, so context assembly **skips the legacy BM25
relevance slice** and reads the engram cascade instead. But the engram importer runs **once, at boot**, and
`memorize`/dream-extraction write **only** to the legacy knowledge store (never `manager.encode` for
fact/procedure/error). Result: everything learned since the last restart is invisible to relevance recall; only
the newest 12 facts survive in the always-on world-model panel (by recency). *This is the "re-derives what it
already stored / write-only memory" failure, on the live path — and it worsens with uptime.*
**Fix:** route `memorize` + dream-extraction fact writes through `manager.encode` when the flag is on, **or**
re-run the idempotent `import_knowledge` on the sleep/dream beat.

### H4 · Salience gate runs every tick but its ranked admission is discarded — `nervous/salience.py:262` (sensory)
The gate is constructed, flag-ON, ingests every tick, consumes the `relevance_set` the core publishes, and
computes the full `admission_bias = bottom_up × top_down × neuromod_gain` with the exploration floor — then the
core reads a **completely separate, unranked** subscription (`AfferentContext.drain_block`) in raw bus-release
order. Top-down goal-relevance, arousal gain, habituation of chatty organs, and the anti-Matthew floor **never
touch what deliberation sees.** *A dark organ wearing a live flag, exactly on the sensory→deliberation seam.*
**Fix:** route `drain_block()` through `gate.admit(max_events)` when enabled (reuse the one subscription to
avoid double-delivery; `admit()` already pass-throughs when flag-off).

### H5 · Privileged control plane is unauthenticated on `0.0.0.0` — `dashboard.py:2518` (safety)
The dashboard binds all interfaces with an empty default token; `reset` (wipe), `git/restore`, `selfedit/apply`,
and `self_guide` (rewrites the directives injected every tick) are reachable by any peer. `/api/control/*`
(stop/pause/sleep) has **no token check at all**, even when a token is set. On the documented Tailscale/LAN
posture this is a real remote kill/wipe/behavioral-hijack surface. *(The "arbitrary RCE via self-edit" angle is
**overstated** — apply only runs pre-staged proposals and `PROTECT_PATHS` fences the safety files — so it's HIGH,
not critical.)*
**Fix (needs an operator decision — lockout risk):** default-bind `127.0.0.1` unless a token is set; auto-generate
a persisted token on first boot; add `_token_ok` to `/api/control/*`. **Documented, not auto-applied** (see §6).

gradient training, tree-search rollouts, external graph DBs, or big API models — so the dominant pattern is

collapse, lossy re-summarization) is the recurring runs-forever failure across every domain.
